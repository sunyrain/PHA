# PHA 设计空间与反向设计计划

日期：2026-07-01

核心问题：PHA 不能只做“水凝胶-蛋白吸附事实库”，还要能支撑反向设计。目标任务应定义为：在给定应用条件下，设计一种水凝胶，使其吸附一组目标蛋白，同时尽量不吸附另一组排斥蛋白，并给出可执行的制备和使用工艺。

## 一、目标任务形式

| 项目 | 定义 |
| --- | --- |
| 输入 | 目标吸附蛋白集合、目标不吸附蛋白集合、应用场景、介质、酸碱度、盐浓度、温度、允许材料、安全/成本/制备约束 |
| 输出 | 候选水凝胶组成、形态、官能团/配体、交联密度或网孔目标、制备路线、吸附/洗脱/再生工艺、证据来源、置信度 |
| 优化目标 | 提高目标蛋白吸附量或选择性；降低非目标蛋白吸附；满足机械、稳定性、生物相容性、可制备性和可放大性要求 |
| 数据单元 | 单蛋白记录用于训练局部响应；多蛋白/竞争吸附记录用于训练选择性；工艺记录用于匹配制备和使用窗口 |

## 二、设计空间分解

| 层级 | 设计变量 | 典型取值或范围 | 对建模的意义 |
| --- | --- | --- | --- |
| 材料骨架 | 天然高分子、合成高分子、复合骨架 | 壳聚糖、海藻酸盐、纤维素、明胶、聚丙烯酰胺、聚乙二醇、聚 N-异丙基丙烯酰胺、两性离子聚合物等 | 决定基础亲水性、可修饰性、生物相容性和工艺窗口 |
| 电荷和官能团 | 阳离子、阴离子、两性离子、中性、混合电荷、疏水基团、亲和基团 | 胺基、羧基、磺酸基、季铵盐、苯基、硼酸、金属螯合基、适配体、肽配体等 | 控制静电吸附、抗污染、选择性识别和 pH 响应 |
| 网络结构 | 交联密度、网孔尺寸、孔径、孔隙率、溶胀率、含水量 | 连续变量，文献常以低/中/高交联或 nm-um 孔径报告 | 控制蛋白进入、尺寸排阻、扩散和容量 |
| 形态和器件 | 块体、薄膜、涂层、微球、纳米凝胶、微凝胶、低温凝胶、膜、柱床、单体柱、支架 | 与应用场景强相关 | 决定批量吸附、表面抗污染、色谱分离或体内/体外使用 |
| 特异性识别 | 印迹模板、亲和配体、离子交换基团、金属配位、硼酸亲和、适配体 | BSA 印迹、Ni-NTA、Cu(II) 桥联、苯丙氨酸/色氨酸配体等 | 提供多蛋白选择性的主要来源 |
| 操作条件 | pH、盐类型、盐浓度、温度、蛋白浓度、接触时间、流速、基质 | 缓冲液、血清、血浆、蛋清、粗提物、细胞培养基等 | 同一材料在不同条件下可能从吸附变成抗吸附 |
| 工艺条件 | 聚合方式、交联方式、接枝、涂覆、冻融/低温聚合、微流控制备、洗脱、再生、灭菌 | 自由基聚合、光聚合、冷冻聚合、表面接枝、层层组装等 | 决定是否能复现实验、放大制备和进入应用 |

## 三、设计空间规模判断

| 估算层级 | 粗略规模 | 说明 |
| --- | ---: | --- |
| 离散材料家族 | 10-30 类 | 只按骨架家族计数，已经有天然、合成、复合、两性离子、响应型等分支 |
| 官能化/配体选择 | 20-100+ 类 | 若加入多配体组合和比例，规模快速膨胀 |
| 结构参数分档 | 5-10 档/参数 | 交联密度、网孔、孔径、溶胀、表面电荷等多为连续变量 |
| 形态和工艺路线 | 8-15 类 | 同一配方做成涂层、微球或低温凝胶柱，结果可能完全不同 |
| 使用条件组合 | 100+ 档 | pH、盐、温度、浓度、基质、流速、时间组合起来远大于单一材料空间 |
| 蛋白组合约束 | 随蛋白数指数增长 | 如果有 N 个候选蛋白，吸附/不吸附集合的组合约为 2^N 级别 |

结论：完整设计空间不是几百个材料，而是“材料 × 官能化 × 网络结构 × 形态 × 工艺 × 使用条件 × 蛋白组合”的组合空间。不能靠穷举，必须把文献数据整理成可约束搜索的形式：先用规则和机理缩小空间，再用模型排序候选方案。

## 四、需要补强的数据类型

| 当前已有 | 还需要补强 | 原因 |
| --- | --- | --- |
| 单一水凝胶-单一蛋白吸附记录 | 多蛋白竞争吸附、选择性、混合基质中的吸附谱 | 目标任务是“吸附一批，不吸附一批”，不是只预测单个蛋白 |
| 吸附容量和抗污染结果 | 选择性因子、目标/非目标吸附比、蛋白 panel 层面的结果 | 需要把材料对蛋白集合的行为做成向量 |
| 材料组成和物性 | 可调设计变量标记，例如哪些变量是作者主动改变的 | 反向设计要知道哪些旋钮能调 |
| 实验条件 | 操作窗口，例如 pH 范围、盐范围、温度范围、流速范围 | 同一水凝胶的选择性常由条件触发 |
| 制备信息 | 工艺可行性、可放大性、后处理、灭菌、循环再生、成本/安全限制 | 最终输出需要匹配工艺，而不是只给材料名 |

## 五、建议新增检索方向

| 新增检索组 | 目的 | 英文概念词 |
| --- | --- | --- |
| Q8 多蛋白竞争吸附和选择性 | 找到真正接近“吸附一批/不吸附一批”的训练数据 | `competitive protein adsorption`, `selective protein adsorption`, `protein adsorption selectivity`, `protein mixture`, `serum protein adsorption`, `plasma protein adsorption`, `protein corona`, `Vroman effect` |
| Q9 配方、制备和工艺窗口 | 抽取可执行制备路线和工艺约束 | `hydrogel formulation`, `hydrogel fabrication`, `crosslinking density`, `gelation`, `cryopolymerization`, `grafting`, `coating`, `regeneration`, `elution`, `scale-up` |
| Q10 反向设计、筛选和机器学习 | 找到数据驱动设计或成组筛选数据 | `machine learning`, `data-driven`, `inverse design`, `screening`, `combinatorial`, `high-throughput`, `hydrogel array`, `protein binding profile` |

建议：这三组先进入专家审阅稿，不立即写入正式 `config.yaml`；专家确认后再跑 count-only probe，避免把文献池扩得太散。

## 六、建议新增字段模块

| 模块 | 字段 |
| --- | --- |
| 设计空间定位 | 应用场景、材料家族、形态家族、功能化策略、网络结构层级、可调变量列表、作者是否系统扫描变量 |
| 蛋白集合结果 | 目标蛋白集合、非目标蛋白集合、竞争蛋白集合、混合基质、蛋白 panel 规模、每个蛋白的吸附向量、选择性排序 |
| 目标约束 | 需要吸附的蛋白、需要排斥的蛋白、最低吸附阈值、最高非目标吸附阈值、选择性阈值、允许操作条件 |
| 工艺匹配 | 推荐制备方法、关键配方比例、交联/接枝/涂覆条件、洗涤、洗脱、再生、灭菌、保存、放大风险 |
| 可实施性评价 | 原料可得性、合成复杂度、是否水相制备、是否温和条件、是否需要有毒单体或金属、是否可循环使用、是否适合目标应用 |
| 反向设计输出 | 候选水凝胶、建议工艺、预期吸附蛋白、预期不吸附蛋白、证据文献、外推风险、置信度、建议验证实验 |

## 七、建模视图建议

| 视图 | 作用 |
| --- | --- |
| 单蛋白响应矩阵 | 行为水凝胶-条件，列为蛋白，值为吸附量、抗污染比例或标签 |
| 蛋白描述符表 | 分子量、等电点、电荷、疏水性、糖基化、构象稳定性、来源 |
| 水凝胶描述符表 | 骨架、官能团、电荷、配体、交联、孔结构、溶胀、形态、工艺 |
| 条件描述符表 | pH、盐、温度、浓度、基质、接触时间、流速 |
| 工艺候选表 | 给定材料的可制备路线、后处理、洗脱再生、适用形态和风险 |
| 反向设计任务表 | 每个任务记录目标吸附集合、排斥集合、应用条件和性能阈值 |

## 八、对当前项目的修改建议

| 优先级 | 修改 |
| --- | --- |
| 必须 | 保留低吸附/抗污染文献，因为它们是“不吸附”目标的负样本来源 |
| 必须 | 增加多蛋白、竞争吸附、血清/血浆/混合蛋白体系的检索和字段 |
| 必须 | 将制备步骤和工艺窗口提升为核心字段，而不是备注字段 |
| 必须 | 在抽取时标记“可调变量”，例如 pH、盐、交联密度、官能团比例、配体密度 |
| 应该 | 单独维护“反向设计请求”和“候选设计输出”两个 schema |
| 应该 | 把文献记录转换成水凝胶-蛋白-条件矩阵，支持多目标优化 |
| 稍后 | 在有 50-100 条高质量标注记录后，再训练初步分类/排序模型 |
